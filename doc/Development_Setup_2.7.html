Install software to access GitHub

There are several options available, some with command-line access and others with Explorer shell integration or stand-alone GUIs. The PyCharm development environment, which we recommend, has built-in access to git.

Install PyCharm Community IDE from https://www.jetbrains.com/pycharm/

This is optional, but is the recommended development environment.

Install Qt 5 Open Source from: http://www.qt.io/download-open-source/

Install Anaconda (Python version 2.7) from https://www.continuum.io/downloads

into a new folder named Anaconda2 in user’s home folder. (Called ~/Anaconda2 below.)

Choose both: Add Anaconda to my PATH and Register Anaconda as my default Python 2.7.

This will automatically add to PATH, for example: C:\Users\...\Anaconda2;C:\Users\...\Anaconda2\Scripts;C:\Users\...\Anaconda2\Library\bin;

If Anaconda3 was previously installed:

Open Control Panel/System/Advanced/Environment Variables/User/PATH and remove any paths that include Anaconda3.

For convenient access, copy the folder:

C:\Users\...\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Anaconda2 (64-bit)

to ~\Desktop\Anaconda2

In this folder, find Anaconda Prompt, run it, and inside the command prompt:

conda install pyqt

  (This installs pyqt4 which is compatible with Qt versions 4 and 5.)

conda install qtconsole

  (needed for the script editor interface; this will also make sure ipython is installed)

conda install -c https://conda.anaconda.org/trent pymsgbox

  (needed by a demonstration plugin)

pip install SWMM5

  (needed to run SWMM5, see https://pypi.python.org/pypi/SWMM5)

keep the shell open if using command-line git for getting source code below.

Get source code from GitHub

Option 1: using PyCharm:

Start PyCharm, click “Check out from Version Control”, choose GitHub.

Log in and choose whether to save your password or use a master password.

Repository URL: https://github.com/USEPA/SWMM-EPANET_User_Interface.git

Directory: choose the parent directory to put the new directory in.

Directory Name: recommend leaving it as the default: SWMM-EPANET_User_Interface

Answer No when asked whether to open the directory.

Choose “Open” on the same splash screen where you chose to check out before.

Open the src folder inside the SWMM-EPANET_User_Interface folder.

If you see a message “Unregistered VCS root detected” choose “Add root”

If you do not see the project files, use View/Tool Windows/Project.

This is the latest release version. For the development version, find “Git: master” in the status bar in the lower right part of PyCharm, click it, choose origin/dev, “Checkout as new local branch”,

Enter name of new branch: dev

OK

Option 2: using git command line:

Change into directory where project folder from repository will be added, then:

git clone https://github.com/USEPA/SWMM-EPANET_User_Interface.git

The above command gets the latest release version. For the development version, also do this:

cd SWMM-EPANET_User_Interface

git checkout dev

To start working on a new feature:

git checkout -b new-feature-name dev

git should respond:  Switched to a new branch 'new-feature-name'

Run PyCharm

Open folder: SWMM-EPANET_User_Interface\src

If you do not see the project files, use View/Tool Windows/Project.

After getting the code using one of the options above, in PyCharm:

File/Settings/Project:.../Project Interpreter: choose ~\Anaconda2\python.exe (press OK)

Wait while PyCharm searches through the Python folders. (Status in bottom bar of PyCharm.)

This can take a long time. You can start looking at code while this completes.

right-click “frmMain.py”in the Project window, choose “Debug frmMain”

Green “Play” button runs, “Bug” button next to it runs in debug mode.

THIS IS THE END OF INSTRUCTIONS FOR NORMAL DEVELOPMENT SETUP

Below are specialized instructions for unusual situations.

Code Documentation Generation

If you are going to generate documentation with Doxygen:

The additional package doxypypy is recommended. (See https://github.com/Feneric/doxypypy.):

pip install doxypypy

Create batch file ~\Anaconda2\Scripts\py_filter.bat containing one line:

doxypypy -a -c %1

Installing development environment for QGIS-integrated version:

First, install QGIS using the OSGeo4W installer from: https://www.qgis.org/en/site/forusers/download.html

Install in C:\OSGeo4W64 (adjust instructions below if not)

Do not install SAGA or GRASS, only QGIS and its libraries.

In Control Panel/System/Advanced/Environment Variables/User, set some variables:

OSGEO4W_ROOT=C:\OSGeo4W64

QGIS=C:\OSGeo4W64\apps\qgis

QGIS_PREFIX_PATH=C:\OSGeo4W64\apps\qgis

PATH=C:\OSGeo4W64\bin;C:\OSGeo4W64\apps\qgis\bin

PYTHONHOME=C:\OSGeo4W64\apps\Python27

PYTHONPATH=C:\OSGeo4W64\apps\qgis\python

Open an OSGeo4W Shell, and in it:

pip install qtconsole

pip install enum34

In PyCharm:

File/Settings/Project: SWMM/Project Interpreter: C:\OSGeo4W64\bin\python.exe (press OK)
